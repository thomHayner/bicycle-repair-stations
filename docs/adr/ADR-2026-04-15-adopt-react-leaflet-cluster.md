## Context

With up to ~25 miles of cached stations rendered at once, marker
overlap at low zoom made it hard to see station density and made
tapping individual stations unreliable. We needed a standard way to
collapse overlapping markers into a single clickable cluster that
grows / shrinks with zoom, while staying compatible with our existing
`react-leaflet` setup and the popup / selection flow.

## Decision

Add `react-leaflet-cluster` (v4.0.0) as a dependency and render
station markers inside its `MarkerClusterGroup` component. This wraps
`leaflet.markercluster` — the de facto standard Leaflet clustering
plugin — and exposes it as a React-friendly component that plays well
with `react-leaflet` v5.

## Alternatives considered

- **Build a custom grid-based clustering layer** — rejected: we'd be
  reimplementing a mature library for no clear benefit.
- **Use `leaflet.markercluster` directly without a React wrapper** —
  rejected: awkward integration with `react-leaflet`'s component tree,
  would require manual lifecycle management.

## Consequences

- **Positive:** standard clustering behavior (grouping, spiderfy,
  animations, zoom-to-bounds) works out of the box. The API is small
  enough to configure inline.
- **Negative:** an additional runtime dependency; we're exposed to the
  library's release cadence and any regressions. Some behavior
  (e.g. spiderfy, cluster click handling) is opaque / not easily
  customizable without digging into Leaflet internals.
- **Follow-ups:** see companion ADRs for how we structure the cluster
  groups and tune the interaction settings.

## Notes

The relevant `.css` asset from the library is imported once in
`src/index.css`:

```css
@import "react-leaflet-cluster/dist/assets/MarkerCluster.css";
```

Cluster icons are generated by `createClusterIcon` /
`createMutedClusterIcon` in `src/lib/leafletConfig.ts`.
